# 电影节排片助手 V1 规划

## 标题与背景

本项目计划基于现有数据源《第十六届北京国际电影节“北京展映”排片表.xlsx》构建一个北影节专用的 Tauri 桌面应用，用于帮助个人影迷完成排片整理、选片决策、冲突处理和日程导出。

当前目录内已经具备一份真实排片 Excel 数据，可作为 V1 的唯一权威数据源。第一阶段不追求通用化，也不先做复杂联网能力，而是优先完成一个离线可用、数据闭环完整、能够直接辅助购票和安排行程的本地助手。

## 产品定位

V1 的产品定位是“个人选片助手”，不是票务平台，也不是纯数据看板。它服务的核心问题是：当用户面对大量场次时，如何更快筛出适合自己的片单，并在预算、时间和赶场成本之间做取舍。

范围边界如下：

- 只支持 `北京展映` 常规放映表。
- 不支持 `无界∞沉浸单元`，后续单独适配。
- 只服务北影节当前这份排片数据，不做跨电影节通用模板。
- 不做多人协作，不做账号体系，不做云同步。
- 不做自动抓取购票平台信息，由用户手动导入 Excel。

默认使用场景如下：

- 单用户、本地使用。
- 离线优先。
- 首次打开后由用户导入 Excel。
- 推荐逻辑基于当前表内字段和用户手动偏好，不依赖外部影片元数据。

## 用户闭环

V1 需要完整覆盖以下闭环：

1. 用户导入排片 Excel。
2. 系统解析并生成影片、场次、影院等结构化数据。
3. 用户通过搜索、筛选、收藏、拉黑等方式初步缩小范围。
4. 用户设置偏好和硬约束，例如预算、可看片日、每日最多场次、价格上限、偏好影院、避开时段。
5. 系统基于规则推荐候选场次，并解释推荐原因。
6. 用户在时间轴视图中查看冲突、替换场次、确认个人片单。
7. 用户导出最终日程，作为购票和观影执行依据。

这个闭环里，推荐不是终点，最终目标是形成一个可执行的个人观影片单。

## 功能模块

### 1. 导入与总览

- 支持选择并导入本地 Excel 文件。
- 展示导入结果摘要，包括影片数、场次数、影院数、日期范围、最近更新时间。
- 提示异常行、缺失字段或未纳入支持范围的数据。

### 2. 场次浏览与筛选

- 支持按影片名、英文名、单元、影院、影厅、活动信息进行搜索。
- 支持按日期、时间段、票价区间、片长区间、单元、影院进行组合筛选。
- 列表项可查看影片基本信息和该影片的全部放映场次。

### 3. 收藏、拉黑与候选管理

- 用户可将影片或具体场次加入“想看”。
- 用户可拉黑不感兴趣的影片或场次。
- 用户可手动指定“必看”“可替代”“不考虑”三类优先级。

### 4. 偏好设置

V1 偏好分为两层：

- 硬约束：可看片日、每日最多场次、总预算上限、单场价格上限、最晚结束时间、默认赶场缓冲时间。
- 软偏好：偏好单元、偏好影院、偏好年份区间、偏好片长区间、优先带映后交流的场次。

默认规则如下：

- 默认赶场缓冲时间为 45 分钟。
- 默认不允许时间重叠场次同时进入最终片单。
- 默认按开始时间、片长和缓冲时间共同计算冲突。

### 5. 推荐引擎

推荐引擎采用“先过滤、后打分、再解冲突”的规则模型：

- 先用硬约束过滤掉不能参加的场次。
- 再根据软偏好、收藏状态、活动信息等对剩余场次打分。
- 最后生成一组不冲突的候选片单，并给出可替换方案。

V1 只做规则推荐，不引入联网元数据，也不引入 AI 对话推荐。

### 6. 时间轴日程视图

- 作为核心可视化页面，按天展示候选和已选场次。
- 清晰标识冲突、缓冲不足、预算超限等风险。
- 支持点击替换冲突场次，快速比较备选方案。
- 右侧展示当日场次统计、预算累计和时间利用率。

### 7. 我的片单与导出

- 按天汇总最终确定的观影片单。
- 展示每场的开始时间、结束时间、影院、影厅、备注信息。
- 支持导出 `.ics` 日历文件和 `.csv` 表格文件。

### 8. 基础统计视图

V1 只保留轻量统计，不将其作为主视图：

- 单元分布。
- 影院分布。
- 每日场次数量。
- 票价分布。

地图路线、交通时间估算不纳入 V1。

## 数据模型与技术方案

### 核心数据模型

`Film`

- `film_id`
- 中文名
- 英文名
- 年份
- 默认片长
- 所属单元

`Screening`

- `screening_id`
- `film_id`
- 放映开始时间
- 票价
- 影院
- 影厅
- 活动信息
- `duration_minutes`

`Venue`

- `venue_id`
- 影院名称
- 可选的地址或备注字段

`PreferenceProfile`

- 日期范围
- 总预算
- 每日场次上限
- 单场价格上限
- 偏好影院
- 避开时段
- 偏好单元
- `must_see` 列表
- `avoid` 列表
- 缓冲时间

`RecommendationResult`

- 候选场次列表
- 总分
- 打分解释
- 冲突对列表
- 可替代方案

`Itinerary`

- 按天分组的已选场次
- 预计花费
- 冲突状态
- 导出元信息

### 技术方案

整体技术栈定为：

- 前端：`Tauri + React + TypeScript`
- 后端能力：`Rust`
- 本地存储：`SQLite`

职责划分如下：

- React 负责页面结构、状态管理、筛选交互、时间轴展示和图表展示。
- Rust 负责 Excel 导入、字段标准化、推荐规则计算、冲突检测、导出 `.ics` / `.csv`。
- SQLite 负责持久化导入数据、用户偏好、收藏状态和最终片单。

建议的 Tauri 命令接口如下：

- `import_schedule(file_path) -> ImportSummary`
- `list_screenings(filters) -> Screening[]`
- `save_preferences(profile) -> PreferenceProfile`
- `generate_recommendations(profile) -> RecommendationResult`
- `save_itinerary(screening_ids) -> Itinerary`
- `export_itinerary(itinerary_id, format) -> ExportResult`

Excel 导入层建议优先针对当前表结构做专用适配，直接读取常规放映表的 A-J 列，跳过标题和提示行。等 V1 跑通后，再扩展为可插拔导入器。

## 非目标、验收标准与后续迭代

### 非目标

以下内容不纳入 V1：

- 联网抓取排片或购票平台链接。
- 自动补充导演、国家、类型、评分等外部影片元数据。
- 地图路线规划和真实通勤时间计算。
- 多人协同选片、分享协作、云端同步。
- 跨电影节通用模板化导入。
- AI 对话式选片顾问。

### 验收标准

满足以下条件时，可认为 V1 达到可用状态：

- 能稳定导入当前 Excel 并跳过标题、提示和异常行。
- 用户能按片名、单元、影院、日期、价格等维度快速筛选场次。
- 推荐结果只返回满足硬约束的场次，并展示明确的推荐原因。
- 时间轴视图能正确检测冲突，并在手动选择冲突场次时及时提示。
- 用户能保存最终片单，并成功导出 `.ics` 和 `.csv`。
- 在约 800+ 场次规模下，浏览、筛选和生成推荐保持流畅。

### 后续迭代方向

V1 之后可按以下顺序扩展：

1. 适配 `无界∞沉浸单元` 特殊排片格式。
2. 引入影院地址与地图能力，估算赶场通勤。
3. 抽象多电影节导入模板，使应用具备复用性。
4. 接入外部影片元数据，增强推荐维度。
5. 增加 AI 对话选片与片单解释能力。

当前阶段先保持单文档规划，不拆分为产品设计、技术设计和任务清单。待开始正式实现后，再进一步拆解详细设计文档和开发任务。
