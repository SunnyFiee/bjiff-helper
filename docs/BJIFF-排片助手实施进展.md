# BJIFF 排片助手实施进展

## 当前状态

项目已经从“前端原型 + Tauri 命令骨架”演进为一套可本地运行、可导入、可保存、可导出的桌面应用雏形。

当前主链路已经打通：

1. 导入 Excel 或回退到内置样本数据
2. 浏览与筛选影片 / 场次
3. 设定偏好和硬约束
4. 生成推荐草案
5. 在时间轴中手动选片和分析冲突
6. 保存片单并导出 `CSV` / `ICS`

## 已完成能力

### 数据与存储

- 已支持从 `北京展映` sheet 导入 Excel。
- 已支持内置样本数据回退。
- 已支持本地 SQLite 持久化：
  - 导入后的数据集
  - 偏好配置
  - 页面状态
  - 豆瓣绑定结果
  - 已保存片单
- 已支持恢复内置样本数据。

### 影片资料

- 已新增影片资料提取脚本 [scripts/extract_film_metadata.py](/Users/sunnyfiee/CODE/CODE_TAURI/bjiff-helper/scripts/extract_film_metadata.py)。
- 已内置 [src/data/bjiff-film-metadata.json](/Users/sunnyfiee/CODE/CODE_TAURI/bjiff-helper/src/data/bjiff-film-metadata.json)。
- 前端加载时会自动把补充资料合并进影片数据。
- 搜索范围已覆盖导演、主演、国家、语言、类型、奖项等资料字段。

### 豆瓣相关

- 已支持打开豆瓣搜索页。
- 已支持手动绑定豆瓣条目 URL 或 subject id。
- 已支持打开已绑定条目。
- 已明确去掉批量联网请求方案，避免不稳定和 `403 Forbidden`。

### 前端界面

- 已完成 `Material UI` 统一重构。
- 已形成桌面端左侧导航 + 主内容区布局。
- 总览、场次浏览、偏好与约束、时间轴、我的片单五页已经成型。
- 总览页已补齐：
  - 数据摘要
  - 排片热度
  - 推荐摘要
  - 资料覆盖情况
  - 最近保存片单
- 顶部主统计卡圆角已收窄，视觉更利落。

### 场次浏览

- 关键词搜索已改为手动触发，不再边输入边筛选。
- 场次浏览列表已改为列式瀑布流，减少高卡片带来的大块空白。
- 单个影片卡片已支持展示：
  - 影片基础资料
  - 豆瓣 / IMDb 评分
  - 推荐草案命中
  - 当前片单命中
  - 当前筛选下的场次列表

### 时间轴

- 每个场次卡片高度按真实时长显示。
- 时间轴左侧已经改为固定视窗高度 + 内部滚动。
- 右侧选片面板已支持：
  - 当前聚焦场次
  - 当前片单衔接反馈
  - 快速操作
  - 影片资料与评分
  - 同片其他场次
  - 附近相关场次
  - 硬约束反馈
  - 当天推荐候补
- 当前轨道优先级已经固定为：
  - 当前片单
  - 已优先
  - 普通场次
  - 已屏蔽
- 已改进分轨算法，按实际渲染高度避让，减少上下重叠影响分析。

### 片单与导出

- 已支持保存当前片单到本地数据库。
- 已支持导出 `CSV` / `ICS`。
- 已支持首页展示最近保存片单。
- 已支持删除单条历史片单与清空历史片单。

### 桌面端壳层

- 当前 macOS 方案已切回原生装饰窗体。
- 目标是优先保证系统级圆角、拖拽和缩放稳定。
- 前端原来自绘的假窗体控制按钮已移除。

## 当前验证结果

已完成的本地验证如下：

- `npm run build` 通过
- `npm run tauri build -- --debug --bundles app` 通过

补充说明：

- 完整 `all` bundle 仍可能在 `dmg` 脚本阶段失败；
- 但 `.app` 主产物已经验证可以正常打出。

## 当前主要不足

- `dmg` 打包链路还不稳定。
- 前端主包体积仍偏大。
- 推荐、导入、导出缺自动化测试。
- 时间轴分析虽然已明显改善，但仍有继续细化空间。
